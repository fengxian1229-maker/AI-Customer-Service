- 每次改動都要使用一個 BT66 編號，例如 `BT66-002`。
- 編號依序增加，不因修改大小跳號。

- 保留原本 LiveChat 收訊與案件流程。
- 加入 Telegram poll overlap 保護，同一個 bot 內部不會同時發兩個 getUpdates。

驗證：
- 已執行 `npm test`，結果為 111 tests passed。

風險：
- 若外部還有其他 bot 或工具搶 Telegram token，快速輪詢不能安全自動關掉對方，只能更快偵測 409。
- Telegram API 呼叫頻率會比原本略高，但仍是同一個 bot 單線輪詢。

## BT66-015｜2026-06-05｜TG 後台自由文字一律轉成禮貌客戶西文

改動目的：
- Lucas 明確要求：後台 TG 回覆主卡時，只要不是之後另做的按鈕字卡，就應把後台自由文字用 LLM 轉成有禮貌、自然的西文回給 LiveChat 客戶。
- 避免 `still processing`、`already on process` 這類後台英文短句原樣或半原樣出現在客戶端。

改動內容：

風險：
